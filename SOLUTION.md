## Що було зроблено та які «пастки» виправлено

### 1. Налаштування Docker та структури проекту

- **Dockerfile:** Створено Docker-образ на базі `python:3.11-slim`, всередину встановлено системний пакет `ffmpeg` (він потрібен бібліотеці `av` для роботи з відео) та інші залежності (`aiortc`, `grpcio`, `behave`).
- **Структура папок:** Щоб Python всередині контейнера не плутався у шляхах до файлів та імпортах, файли сервера (`server.py`, `sample.mp4.enc`) скопійовано прямо в корінь робочої директорії, а тести — поруч у папку `features/`.

### 2. Генерація gRPC-коду

Генерацію файлів `signaling_pb2.py` та `signaling_pb2_grpc.py` додано прямо у процес збірки Docker. Тепер вони створюються автоматично командою `python -m grpc_tools.protoc`.

### 3. Виправлення помилок у коді (Пастки)

- **Пастка 1 (Падіння сервера):** Виправлено шлях запуску скрипту в інструкції `CMD`. Раніше контейнер падав відразу після старту, бо не міг знайти файл `server.py`.
- **Пастка 2 (Помилка "different loop" у тестах):** gRPC в Python дуже чутливий до асинхронних потоків (`asyncio`). Спочатку тести падали, бо канал зв'язку створювався в одному потоці, а самі запити йшли в іншому. Рішення — перенесено створення gRPC-каналу (`async with grpc.aio.insecure_channel`) всередину кожного окремого кроку тесту. Тепер кожен крок сам створює підключення, робить запит і закриває його.

### 4. Автоматизація тестів (Behave)

Усі 3 сценарії з файлу `video.feature` тепер працюють:

1. **Тест відео (WebRTC):** Клієнт створює Offer, сервер успішно розшифровує відеофайл ключем AES, підключає трек і повертає коректний AnswerMessage.
2. **Тест чату (Unary):** Перевірено відправку звичайного повідомлення. Сервер повертає точний ехо-дублікат тексту.
3. **Тест стріму (Stream):** Перевірено отримання потоку повідомлень від сервера. Клієнт успішно чекає і зчитує всі 3 повідомлення (тест триває близько 3 секунд, що підтверджує роботу затримок на сервері).

## Інструкція для швидкого запуску тестів

Виконайте ці чотири команди в терміналі по черзі:

```bash
# 1. Збираємо Docker-образ
docker build -t my-grpc-server .

# 2. Видаляємо старий контейнер, якщо він залишився
docker rm -f grpc_server

# 3. Запускаємо сервер у фоні
docker run -d -p 50051:50051 --name grpc_server my-grpc-server

# 4. Запускаємо Behave-тести прямо всередині контейнера
docker exec -it grpc_server behave features/video.feature
```
